Raport Proiect: Books on Web (BoW)

Proiect realizat de: Alexandru Raducanu, Sacaleanu Stefan-Bogdan

1. Specificarea Cerintelor Software (SRS)

Aceasta sectiune detaliaza cerintele pentru aplicatia BoW, urmand o structura inspirata de IEEE System Requirements Specification Template.

1.1 Introducere

Acest document defineste cerintele pentru aplicatia "Books on Web" (BoW), un sistem web destinat gestionarii cartilor, urmaririi progresului de lectura si interactiunii in grupuri sociale.

1.2 Scopul Produsului

BoW este o platforma software al carei scop este de a centraliza experienta de lectura a utilizatorilor. Obiectivele principale includ oferirea unui catalog de carti usor de navigat, unelte pentru monitorizarea activitatii de lectura si facilitarea formarii de comunitati.

1.3 Clase de Utilizatori si Caracteristici

  • Utilizator Standard: Orice persoana care se inregistreaza in aplicatie. Poate cauta carti, salva progresul, lasa recenzii si interactiona in grupuri.
  • Administrator: Un utilizator cu privilegii speciale, definit de atributul is_admin in baza de date. Are toate drepturile unui utilizator standard, plus capacitatea de a adauga, edita si sterge carti din sistem.

2. Arhitectura de Ansamblu (Modelul C4)

Arhitectura aplicatiei este descrisa in detaliu in documentatia proiectului si este prezentata aici sub forma modelului C4.

2.1 Contextul Sistemului

La nivel de context, sistemul BoW este o aplicatie web care interactioneaza direct cu utilizatorii sai (Standard si Administratori) si se conecteaza la un sistem extern pentru servicii de geolocatie, API-ul Nominatim.

2.2 Containere

Sistemul este alcatuit din urmatoarele containere logice care colaboreaza:

2.3 Componente

Containerul Backend este modularizat in componente MVC:

  • Controllere (controllers/): Punctul de intrare pentru logica. ControllerAuth gestioneaza autentificarea, ControllerFeed gestioneaza catalogul de carti, iar ControllerGroup se ocupa de grupuri. Un front controller (index.php) directioneaza cererile.
  • Modele (models/): Responsabile cu interactiunea cu baza de date prin PDO si prepared statements.
  • View-uri (views/): Clase PHP care randeaza HTML, populand template-uri si securizand output-ul cu htmlspecialchars.

2.4 Etape de Dezvoltare

  1. Fundatia Proiectului: Definirea schemei bazei de date (db_script.sql) si configurarea structurii de baza MVC si a dependintelor cu Composer (composer.json).
  2. Implementarea Nucleului Functional: Crearea sistemului de autentificare (ControllerAuth, ModelLogin, JWT) si afisarea catalogului de carti (ControllerFeed, ModelFeed).
  3. Extinderea Functionalitatilor: Adaugarea modulului de grupuri (ControllerGroup, ModelGroup) si a API-ului pentru administrare (ControllerApiFeed).
  4. Imbunatatiri Client-Side: Implementarea interactivitatii cu JavaScript, cum ar fi filtrarea AJAX (feed_filters.js) si geolocatia (geolocation.js).

3. Proiectarea Interfetei Web

3.1 Motivatia Designului

Designul interfetei urmareste un stil minimalist si functional, axat pe lizibilitate si usurinta in utilizare. Paleta de culori este rece, cu accente de albastru, avand ca scop crearea unei experiente de navigare calme si placute. Grila de carti (.book-list) foloseste un layout flexibil pentru a se adapta eficient la continut variabil.

3.2 Design Responsiv

Interfata este complet responsiva, utilizand CSS Media Queries pentru a se adapta la diferite dimensiuni de ecran. La rezolutii sub 1300px, bara de navigare se transforma intr-un meniu de tip "hamburger" controlat de nav.js, pentru a maximiza spatiul disponibil pe ecranele mobile. Layout-ul general trece de la o structura multi-coloana la una single-column pentru a asigura o experienta optima.

4. Structura si Provenienta Datelor

4.1 Modelarea Datelor

Structura datelor este relationala si este definita in db_script.sql.

  • Entitati Principale: users, books, groups.
  • Relatii: Tabelele de legatura precum user_book_progress si group_members modeleaza relatiile many-to-many.
  • Integritate: Baza de date foloseste triggere (ex: trg_validate_username_length) pentru a impune reguli de business direct la nivel de date, asigurand consistenta indiferent de aplicatie.

4.2 Provenienta Datelor

  • Date Initiale: Catalogul initial de carti este populat printr-un script de inserare in db_script.sql.
  • Date Generate de Utilizatori: Majoritatea datelor dinamice (recenzii, evaluari, progres, grupuri) sunt generate direct de catre utilizatori prin intermediul interfetei web.
  • Date din Servicii Externe: Informatiile despre bibliotecile din apropiere sunt obtinute in timp real de la API-ul public Nominatim, asa cum este implementat in functia findLibrariesNearby din ModelFeed.php.
